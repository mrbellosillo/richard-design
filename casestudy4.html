Aura Finance: A Visual Savings Experience

Designed a premium mobile savings application that redefines the personal finance experience through atmospheric UI and emotional feedback. The focus was on reducing financial anxiety, implementing modern glassmorphism patterns, and creating a rewarding goal-tracking system for Gen Z users.

ROLE
UI/UX Designer
DURATION
5 Days
TOOL
Figma
Jump to final screens
THE PROBLEM

Traditional banking and investment applications often feel intimidating and cluttered, leading to "financial anxiety" among younger users. The lack of visual reward and the presence of complex jargon make it difficult for students and recent graduates to maintain consistent saving habits.

USERS NEED TO:
  • Track savings goals without feeling overwhelmed by data
  • Visualize progress in a motivating and modern interface
  • Perform "Quick Save" actions with minimal friction
  • Receive clear emotional feedback upon successful transactions
→ The main challenge was designing a high-end, "lifestyle-focused" interface that balances advanced visual effects like glassmorphism with strict financial clarity.
GOALS & OBJECTIVES
  • Humanize the savings process through "Aura" visual metaphors
  • Implement a glassmorphism-based UI for a premium, modern feel
  • Simplify complex financial data using a structured neutral scale
  • Reduce cognitive load during the fund-transfer process
  • Provide "User Delight" through celebratory success states
UNDERSTANDING THE USER & WORKFLOW
User Persona
To design an app that truly reduces financial anxiety, I first defined the core user. Maya, a hesitant starter, represents the Gen Z user who is motivated by goals but overwhelmed by traditional banking layouts. Defining her pain points allowed me to prioritize "Atmospheric UI" as a way to build comfort and trust.
User Flow

I mapped out a "Quick Save" journey to ensure the path from the Dashboard to the Success state was as frictionless as possible.

  1. Entry: User views goal progress on the Dashboard.
  2. Action: User selects a pre-set amount (+$20) in the Goal Detail.
  3. Verification: System confirms identity via FaceID.
  4. Completion: User receives immediate visual and emotional feedback.
KEY INSIGHT
“Financial interfaces are most effective when they reduce anxiety and replace technical complexity with emotional rewards.”
PROCESS
1
Layout & Structure
Focused on a "Bento-style" layout for the dashboard to organize different data types (Balance, Quick Actions, Goals). Information was grouped to prioritize the "Total Balance" while keeping "Active Goals" easily accessible via horizontal scrolling.
2
Interaction Design
Designed intuitive touch targets for the "Quick Save" buttons. The FaceID verification was implemented as a blurred glass overlay to maintain user context while providing a sense of security and system feedback.
3
Consistency
Established a rigorous 10-step Neutral scale tinted with the primary green hue to ensure all grays felt harmonious. Maintained consistent border-radii (24px) and 1px stroke gradients across all glass components.
4
Refinement
Refined the typography scale using "Outfit" to ensure numerical clarity. Adjusted the letter-spacing (tracking) for large display numbers and utilized tabular figures to prevent "jitter" during value changes.

REFLECTION

This project strengthened my ability to handle advanced visual styling while maintaining accessibility. It taught me that UI "decoration," when used correctly (like the Aura glows), serves a functional purpose by reducing data anxiety. I learned the technical nuances of designing for dark mode and the importance of using tinted neutrals for a premium finish.
