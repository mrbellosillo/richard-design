Voucher & Promo System

Designed the voucher, promo, and offer card experience for a ride-hailing application. The focus was on improving interaction clarity, aligning the interface with the booking flow, and maintaining consistent UI patterns across components.

THE PROBLEM

The voucher system was accessible during the booking process, but the interface did not clearly support voucher selection. Key actions and interaction patterns were not aligned with the user’s current step, which could lead to confusion.

USERS NEED TO:
  • Apply vouchers quickly during booking
  • Understand which voucher is currently selected
  • Manually enter promo or gift codes
  • View complete voucher details without friction
→ The main challenge was designing a system that fits seamlessly into the booking flow while keeping interactions clear and consistent.
GOALS & OBJECTIVES
  • Align voucher actions with the booking process
  • Improve clarity of voucher selection and states
  • Maintain consistent visual and interaction patterns
  • Reduce unnecessary user actions (extra taps)
  • Ensure system limitations are clearly communicated
UNDERSTANDING THE WORKFLOW
  1. Analyzed how vouchers are used within the booking process
  2. Identified mismatches between user intent and system actions
  3. Reviewed existing booking screens for consistency
  4. Focused on reducing friction during voucher selection
KEY INSIGHT
“Interfaces should always match the user’s current context — especially during task-focused flows like booking.”
PROCESS
1
Layout & Structure
Organized the voucher cards and detail screens to support quick scanning and easy selection. Focused on clarity of actions and minimizing friction within the booking flow.
2
Interaction Design
Defined interaction states across multiple components such as voucher cards, input fields, and buttons to ensure clear and consistent user feedback throughout the experience.
3
Consistency
Standardized visual elements such as colors, spacing, and component styles across multiple screens to maintain a cohesive and predictable interface.
4
Refinement
Adjusted spacing, hierarchy, and visual emphasis (such as borders and indicators) to improve usability and make selections more noticeable.

REFLECTION

This project strengthened my ability to design within an existing system while considering both user flow and developer constraints. It highlighted the importance of aligning interface actions with user intent, especially in task-driven scenarios like booking.

Working with feedback from both design and development also reinforced the value of iteration and collaboration in creating practical and usable solutions.
